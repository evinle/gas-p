# Triage Labels

The skills speak in terms of five canonical triage roles. This file maps those roles to the actual label strings used in this repo's issue tracker (GitHub Issues).

| Canonical role    | Label in this tracker | Meaning                                  |
| ------------------ | ---------------------- | ----------------------------------------- |
| `needs-triage`     | `needs-triage`          | Maintainer needs to evaluate this issue   |
| `needs-info`       | `needs-info`            | Waiting on reporter for more information  |
| `ready-for-agent`  | `ready-for-agent`       | Fully specified, ready for an AFK agent   |
| `ready-for-human`  | `ready-for-human`       | Requires human implementation             |
| `wontfix`          | `wontfix`               | Will not be actioned                      |

Category roles (`bug`, `enhancement`) map 1:1 to the pre-existing `bug` and `enhancement` GitHub labels.

All five state labels above were created fresh in this repo on 2026-07-04 — none existed before triage tooling needed them.
